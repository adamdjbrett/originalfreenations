1. Background

1.1 The Big Picture

Original Nations is a site publishing research on indigenous peoples' liberation, the Doctrine of Christian Discovery, and Federal Indian Law. It originally ran on WordPress with the "Coup-2" theme. The task was to migrate to a static site builder (Eleventy v3) while adopting the visual design of the Headline theme from the Ghost publishing platform — a modern, clean, card-based layout optimized for content discovery.

The Headline theme features a "stacked" header (logo centered above a horizontal nav bar), a home page organized as a series of "topic grids" (Latest posts, then category/tag sections), and a dark-background footer with subscribe call-to-action, navigation links, social icons, and copyright. Every section uses a specific class hierarchy from Ghost's CSS conventions.

1.2 The Existing System

Before this change, there was a static Eleventy site with:

  • 57 posts converted from WordPress XML via wordpress-export-to-markdown, stored as src/content/posts/{slug}/index.md with YAML frontmatter.
  • 5 pages (About, Contact, News, Support Our Work, home-2) in src/content/pages/.
  • Per-post images in src/content/posts/{slug}/images/, passthrough-copied to _site/{slug}/images/.
  • Baseline plugin (@apleasantview/eleventy-plugin-baseline v0.1.0-next.44) providing SEO graph, virtual directories, head injection, and asset pipeline.
  • screen.css — the compiled, postcss-processed CSS from the Headline theme's gulp build pipeline. ~25KB minified.
  • Templates using Nunjucks (.njk) with markdownTemplateEngine: "njk" so Markdown content is rendered inside Nunjucks layouts.

1.3 The Target: Headline Ghost Theme

The Headline theme's architecture centers on a few key patterns:

Class Hierarchy Convention. Every section uses a gh- prefixed naming system: gh-head, gh-foot, gh-card, gh-topic, gh-article. These are not just cosmetic — the CSS grid layout depends on this specific nesting to create the two-column card grid with the first card spanning full width.

The home page has this structure:

gh-topic gh-topic-grid: "Latest" (first card .large, then 6 regular cards, "Show more" link)
gh-topic gh-topic-grid: "Environment" (same pattern)
gh-topic gh-topic-grid: "Culture" (same pattern)
gh-topic gh-topic-grid: "Sports" (same pattern)

1.4 A Note for Beginners

If you're new to static site generators: Eleventy (11ty) takes templated content files (Markdown with YAML frontmatter) and renders them through template engines (Nunjucks, Liquid, etc.) into plain HTML. No database, no server-side runtime — just files. The .eleventy.js config file defines collections (groups of content), filters (transformations for templates), and passthrough copies (files like images that should be copied verbatim to the output directory).

Nunjucks is a JavaScript templating language similar to Jinja2 (Python) or Liquid (Ruby). It uses {% %} for logic tags and {{ }} for variable output. Filters like | default([]) | join(", ") let you transform data inline.

2. Intuition

2.1 The Core Idea

The Headline theme's visual identity is built on a grid of cards organized by topic. Each topic section has exactly the same structure: a heading, a two-column card grid where the first card spans the full width, and a "Show more" link. Recreating this in Eleventy means mapping the Ghost theme's Handlebars partials to Nunjucks templates while preserving every class name and wrapper, so the existing CSS works without modification.

2.2 How the Card Grid Works

Here's the key CSS grid pattern. The container is a two-column grid. The first card gets class="large", which sets grid-column: 1/3, making it span both columns. The remaining cards flow naturally into the two columns.

The CSS driving this:

.gh-topic-grid .gh-topic-content {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 32px 40px;
}
.gh-topic-grid .gh-card.large {
  grid-column: 1 / 3;  /* spans both columns */
}

2.3 Three Changes that Fixed Visual Bugs

Bug 1: Nav text invisible. The CSS contained body:not(.is-dropdown-loaded) .gh-head-menu .nav>li{opacity:0} — a rule intended for Ghost's JavaScript dropdown animation. Since Ghost JS never runs on a static site, the nav was permanently invisible. Fix: change opacity:0 → opacity:1.

Bug 2: Broken accent color. The CSS used var(--ghost-accent-color) in dozens of places (links, buttons, tag labels, article borders) but never defined the variable. Fix: add --ghost-accent-color: #2563eb to :root.

Bug 3: Cover images in wrong path. The WordPress export placed cover images inside per-post images/ subdirectories. The template needed to reference images/{filename} relative to the post URL, which resolves correctly because the passthrough copy puts the entire images/ folder at _site/{slug}/images/.

2.4 Composing Tag Sections Dynamically

Posts in the WordPress export don't use categories — only tags (100 unique tags across 57 posts). The top 3 tags by count were: united-states (10 posts), supreme-court (9), christian-europeans (7). A custom Nunjucks filter filterByTag was added to Eleventy so the home page can iterate over a list of tag names and render a grid section for each:

{% for tag in topTags %}
{% set taggedPosts = collections.posts | filterByTag(tag) %}
<section class="gh-topic gh-topic-grid">
  <h2 class="gh-topic-name">
    <a href="/tag/{{ tag }}/">{{ tag | split("-") | join(" ") | title }}</a>
  </h2>
  ...
</section>
{% endfor %}

The split("-") | join(" ") | title filter chain converts "united-states" into "United States" for display.

        </figure>
        {% endif %}

          <header class="gh-card-header">
            <h3 class="gh-card-title">{{ post.data.title }}</h3>
          </header>
          <footer class="gh-card-footer">
            <span class="gh-card-author">
              {{ post.data.tags | default([]) | join(", ") }}
            </span>
            <time class="gh-card-date" datetime="...">...</time>
          </footer>
        </div>
      </a>
    </article>
    {% endfor %}
  </div>

3.4 New Custom Filters

File: .eleventy.js

Two filters were added to enable dynamic tag sections:

eleventyConfig.addFilter("filterByTag", (posts, tag) =>
  (posts || []).filter(p =>
    p.data && p.data.tags && p.data.tags.includes(tag)
  )
);
eleventyConfig.addFilter("split", (str, sep) =>
  (str || "").split(sep)
);
Why filterByTag instead of a collection? Tag collections would need to be defined at build time in .eleventy.js. Using a filter is more flexible — it works with any tag name at render time, doesn't require config changes when tags change, and keeps the logic in the template where it's easy to understand.
